# LLD (Low-Level Design)

## What belongs here
One note per object-oriented / class-design problem — e.g. "Design a Parking Lot," "Design an Elevator System." Use `Templates/LLD Template.md` for every new note.

## How to use it
1. Copy `Templates/LLD Template.md` into this folder.
2. Rename it to the problem you're designing.
3. Sketch classes and relationships before writing code — the template's order (Requirements → Classes → Relationships → Design Patterns) forces that discipline.
4. Note which design pattern you reached for and why — interviewers care more about the "why" than the UML.

## Naming convention
Plain English, named after the problem: `Design a Parking Lot.md`, `Design an Elevator System.md`.
