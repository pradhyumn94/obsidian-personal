# HLD (High-Level Design)

## What belongs here
One note per system design problem you practice — e.g. "Design a URL Shortener," "Design Twitter's News Feed." Use `Templates/HLD Template.md` for every new note.

## How to use it
1. Copy `Templates/HLD Template.md` into this folder.
2. Rename it to the system you're designing.
3. Time yourself (45-60 min for a full rep) and fill it in as you go, not after the fact — the template's sections (Requirements → Scale → Architecture → ... → Trade-offs) mirror the order you should think in during a real interview.
4. Come back before interviews and skim the "Revision Notes" section across all your HLD notes.

## Naming convention
Plain English, named after the system: `Design a Rate Limiter.md`, `Design Twitter Feed.md`. No numbering, no prefixes.
