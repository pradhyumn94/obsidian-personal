2. Object oriented design principles (SOLID)
	1. SRP - Single Responsibility principle - one reason to change
	
	2. OCP - Open closed principle - open for extension and closed for modification
	
	3. LSP - Liskov Substitution principle - A subclass should be usable anywhere its parent class is expected without breaking the program's correctness.
	   
	4. ISP - Interface segregation principle - clients should not be forced to depend on methods they do not use. Instead of one large "fat" interface, create smaller, focused interfaces.
	   
	5. DIP - Dependency Inversion principle  - High-level modules should not depend on low-level modules. Both should depend on abstractions. Abstractions should not depend on details; details should depend on abstractions.
